# ADR 0002 — 외부 도구 연동을 두지 않는다 (오프라인·로컬 정체성 유지)

| 항목 | 내용 |
|---|---|
| 상태 | Accepted |
| 결정일 | 2026-07-22 |
| 관련 | [PRODUCT.md](../PRODUCT.md) 비목표, [ADR 0001](0001-goodz-provenance.md) |

## 배경

Cycle을 에픽-이슈-태스크 트리로 보게 만든 뒤, "Notion·Jira·Figma와 연동해 거기 내용도 가져와 반영하자"는 제안이 나왔다. 실제 팀 도구의 계획·디자인을 Beacon 화면에 끌어오면 편의성은 오른다.

## 결정

**Beacon은 외부 SaaS(Notion·Jira·Figma·GitHub API 등)와 실시간 연동하지 않는다.** 진실의 원천은 **프로젝트 폴더의 파일과 로컬 Git**으로 유지한다.

## 이유

두 개의 제품 정체성과 정면으로 충돌하기 때문이다.

1. **오프라인·로컬 우선 ↔ 외부 API는 네트워크·인증을 요구한다.** OAuth 토큰, API 키, 네트워크 왕복이 들어오는 순간 "인터넷 없이 폴더만 있으면 동작한다"는 성질이 깨진다. [PRODUCT.md](../PRODUCT.md) 비목표에 `계정, RBAC, Cloud 동기화`가 이미 명시돼 있다.

2. **"파일·Git이 진실" ↔ 외부 데이터를 가져오면 판정 근거가 흐려진다.** Beacon의 Health·Gate·Timeline은 "폴더에서 실제로 확인 가능한 것"만 근거로 삼는다(제품 원칙: 모든 신호는 근거·출처·다음 행동을 가진다). Notion의 태스크 상태나 Jira 티켓을 근거로 섞으면, 판정이 "폴더 밖의, 검증 불가능한 상태"에 의존하게 되어 원칙이 무너진다.

즉 외부 연동은 "v0.2의 개선"이 아니라 **Beacon을 다른 제품(팀 협업 허브)으로 바꾸는 전략 전환**이다. 지금은 그 전환을 하지 않는다.

## 제외 (하지 않는 것)

- Notion·Jira·Figma·Linear 등과의 OAuth/API 실시간 연동
- 외부 티켓·태스크 상태를 Health·Gate·Timeline의 판정 근거로 사용
- 계정·토큰·클라우드 동기화 계층

## 나중에 다시 연다면 지킬 원칙

이 결정을 뒤집을 때는 아래 절충 중 하나로, **정체성을 깨지 않는 범위**에서만 검토한다.

- **로컬 export 기반 관찰**: Figma/Notion을 사용자가 로컬로 내보낸 파일(예: `.md`, `.pdf` export)을 폴더에 두면, Beacon은 그것을 기존 산출물처럼 **읽기만** 한다. 네트워크 불필요, "폴더가 진실" 유지.
- **읽기 전용·판정 제외**: 정 연동한다면 외부 데이터는 "참고 링크"로만 표시하고, Health·Gate 근거로는 **절대 쓰지 않는다.** 판정은 여전히 파일·Git만으로 한다.
- **명시적 경계**: 어떤 경우에도 외부에서 온 정보는 화면에서 출처가 "외부"임을 분명히 표시해, 로컬 관찰 결과와 섞이지 않게 한다.

## 결과

Beacon은 "내 컴퓨터의 프로젝트 폴더 하나를 정직하게 비추는 로컬 도구"라는 정체성을 유지한다. 팀 도구 연동이 필요한 사용자는 그 도구를 계속 쓰고, Beacon은 그와 경쟁하지 않는다 — Beacon의 가치는 **실제 코드·Git이 계획과 얼마나 맞는지 자동으로 비추는 것**에 있다.
