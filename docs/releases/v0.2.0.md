# Beacon v0.2.0

실제 외부 쇼핑몰 저장소 `saleor/storefront`를 Beacon으로 관찰하고 원본 README, ADR, 테스트와 Git log를 대조해 관찰 정확도를 개선했습니다.

## 설치

```bash
pnpm add --save-dev https://github.com/dayainow/beacon-project-os/releases/download/v0.2.0/dayainow-beacon-0.2.0.tgz
pnpm exec beacon init
pnpm exec beacon open
```

Node.js 22.13 이상이 필요합니다.

## 주요 변경

- `project` 핵심 산출물과 `support` 자동화·지원 문서를 구분합니다.
- `skills`, `.agents`, `.claude`, `.cursor`, `.github` 문서는 Gate 근거에서 제외합니다.
- Dashboard에서 핵심 산출물과 지원 문서 수를 따로 보여줍니다.
- `product-pdp.md` 같은 상품 기능 문서를 PRD로 오인하지 않습니다.
- 일반적인 Fix·Document·Migrate commit도 Timeline 의미로 분류합니다.
- Health 기준이 하나라도 부족하면 `attention`으로 표시합니다.

## 외부 검증 결과

Saleor Storefront 기준 Health 과대평가를 100%에서 80% attention으로 정정했고, 102개 문서를 핵심 14개와 지원 88개로 분리했습니다. P0 기획 근거 오탐을 제거하고 P1 설계 출처는 실제 `docs/adr/*`로 연결했습니다. Timeline의 단순 `change` 분류는 20개 중 18개에서 4개로 줄었습니다.

상세 기록은 `docs/evaluations/0001-saleor-storefront.md`에서 확인할 수 있습니다.
