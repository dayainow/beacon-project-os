# Beacon v0.3.0

대시보드를 실무 수준으로 다시 디자인하고, 실제 팀 저장소에서 신뢰할 수 있도록 판정 안정성·데이터 정확성·견고성을 강화했습니다.

## 설치

```bash
pnpm add --save-dev https://github.com/dayainow/beacon-project-os/releases/download/v0.3.0/dayainow-beacon-0.3.0.tgz
pnpm exec beacon init
pnpm exec beacon open
```

Node.js 22.13 이상이 필요합니다. 상세 사용법은 [GUIDE](../GUIDE.md).

## 대시보드 개편

- 밝은 라이트 톤의 떠 있는 카드 레이아웃과 디자인 토큰, 고급 시스템 폰트 스택으로 전면 재디자인.
- 개요: 진행 단계 도넛, Health 바, 활동량 그래프. 중복 지표를 정리하고 정보 위계를 잡음.
- 단계: 진행바·상태 아이콘·단계별 요구 힌트. 각 단계 카드를 클릭하면 그 단계 상세로 전환.
- 산출물: 종류별 그룹화 + 어느 단계(P0–P4) 근거인지 표시. 텍스트 문서(.md/.mdx/.txt)를 클릭해 대시보드 안에서 내용 보기.
- 히스토리: 일자별 타임라인 피드, Cycle을 에픽–이슈–태스크로 펼쳐 보기, commit 작성자 표시.
- 사용 가이드 화면 내장, 문구를 시스템 용어에서 이해하기 쉬운 말로 정리.

## 실무 저장소 대응

- `.gitignore`를 존중해 빌드 캐시·생성 파일·비밀 폴더를 스캔에서 제외.
- commit 관찰 범위를 넓혀(20 → 500) 판정이 시점에 따라 흔들리지 않게 함.
- commit 작성자(author)를 Timeline·이슈에 표시.
- History가 표시 상한을 넘는 Timeline 이벤트도 누락 없이 누적하도록 수정.
- 대형 레포(zod, 583파일·2927commit) 스캔 267ms 실측(`docs/evaluations/0002`).

## 견고성

- `.beacon/beacon.db`가 손상되거나 스키마 버전이 다르면 손상 파일을 백업하고 자동으로 재생성. 대시보드가 통째로 멈추지 않음.
- GitHub Actions CI로 main·PR마다 빌드·린트·테스트·패키지 검증 자동 실행.

## 결정 기록

- 외부 도구(Notion·Jira·Figma) 연동을 두지 않고 오프라인·로컬 정체성을 유지(ADR 0002).
- P0–P4를 고정 표준으로 유지하고 적용 대상을 "제품 개발 프로젝트"로 명확히 함. 라이브러리에서 Health가 낮은 것은 결함이 아니라 "대상 아님" 신호(ADR 0003).
